### Функционал бота

- Панель в канале `PANEL_CHANNEL_ID`: кнопка **«Подать заявку»** → модалка (ранг, имя, фамилия).
- Заявка в БД, в `MOD_CHANNEL_ID` — embed с **Approve** / **Reject**.
- Одобряют только участники с **хотя бы одной** ролью из `APPROVER_ROLE_IDS`.
- **Approve** → модалка «способ приёма» → все роли из `TARGET_ROLE_IDS`, ник по шаблону, запись в `AUDIT_CHANNEL_ID` + пинг `ACADEMY_ROLE_ID`.
- **Reject** без выдачи ролей.
- Защита от повторной обработки заявки.
- По cron — сводка в `DAILY_STATS_CHANNEL_ID` (одобрения за «вчера» в `DAILY_STATS_TIMEZONE`).
- Попытка очистить slash-команды при старте; при ошибке доступа процесс не падает.

### Права бота (permissions)

- Manage Roles, Manage Nicknames  
- Отправка сообщений, embed, чтение истории в задействованных каналах  
- Инвайт: `bot` + желательно `applications.commands`  
- Роль бота **выше** выдаваемых ролей.

### Intents

- **Server Members Intent** — да.  
- Message Content — нет. Presence — нет.

### Развёртывание

1. `npm install` → `npx prisma generate` → `npx prisma migrate deploy`  
2. `.env` из `.env.example` (секреты не в git)  
3. `npm run build` → `npm start`  
4. На проде по регламенту: **PostgreSQL** — см. [`MIGRATION_POSTGRES_RU.md`](./MIGRATION_POSTGRES_RU.md) и [`docker-compose.yml`](../docker-compose.yml).

### Логи для администрации (регламент п. 7.3)

- Консоль: структурированный вывод `pino`.  
- Опционально: `LOG_TO_FILE=true` → JSON-строки в `LOG_FILE_PATH` (по умолчанию `logs/chopbot.log`), файл **не** в git.
